UTILISATION DU SCRIPT JAPONAIS

Le script peut être exécuté à l’aide de la commande suivante : 

bash script_jp.sh /chemin/vers/les/urls.txt japonais

Le traitement du japonais est complexe : c’est une langue qui utilise deux syllabaires et des sinogrammes, n’emploie pas d’espaces entre les mots et ne repose pas sur l’alphabet latin.

Dans ce contexte, les méthodes traditionnelles de comptage fondées sur la séparation par espaces, telles que la commande wc -w, sont inadaptées. Afin de garantir un traitement correct des caractères japonais, l’encodage UTF-8 est explicitement forcé dès le début de l’exécution du script. Le repérage des occurrences repose alors sur une approche par motifs de caractères, à l’aide d’expressions régulières, plutôt que sur une segmentation lexicale complète.

Pour extraire et compter les occurrences du mot cible, la commande grep -o est utilisée afin d’isoler chaque correspondance. Le nombre total d’occurrences est ensuite obtenu en comptant le nombre de lignes produites à l’aide de wc -l. Cette méthode permet de contourner l’absence de séparation lexicale explicite en japonais, tout en évitant le recours à des outils de segmentation morphologique plus complexes.

Avant toute exécution, le script vérifie la présence des outils nécessaires à son fonctionnement. Le programme curl est utilisé pour télécharger les contenus distants et récupérer les codes de réponse HTTP. L’outil pup permet l’extraction du texte à partir des pages HTML à l’aide de sélecteurs CSS, tandis que pdftotext est employé pour convertir les documents PDF en texte brut.

Les contextes sont ensuite extraits autour de chaque occurrence du mot cible à l’aide d’une expression régulière capturant un nombre défini de caractères avant et après le mot. Ces extraits servent à construire le concordancier et permettent d’observer les occurrences dans leur environnement immédiat.

Les fichiers générés ont des rôles complémentaires : les textes bruts contiennent le contenu original, les pages HTML facilitent la lecture des lignes pertinentes, et le concordancier rassemble l’ensemble des contextes. Le tableau HTML final synthétise toutes les informations par document.

Note

Il convient de noter que les fichiers texte générés ne comportent pas de déclaration explicite d’encodage. Contrairement aux pages HTML, ils ne contiennent pas de balise <meta charset="UTF-8">, ce qui peut entraîner des problèmes d’affichage lors de leur ouverture directe dans un navigateur. Afin de respecter les exigences de la consigne, ces fichiers ont été conservés au format texte brut.

UTILISATION DU SCRIPT RUSSE

Le script peut être exécuté à l’aide de la commande suivante : 

bash script_ru.sh ../URLs/urls-ru.txt ../tableaux/tableau-ru.html

et il accepte deux arguments (leur présence est vérifiée au début du scripte) : le chemin vers le fichier avec une liste des URLs et le chemin vers le tableau html des résultats.

Lors du travail avec la langue russe, nous nous sommes limités au traitement de pages HTML.

Dans ce programme, la commande curl est utilisée pour télécharger la structure HTML des pages web listées danq le fichier urls-ru.txt. Immédiatement après, une vérification de l’encodage du texte de la page web est effectuée. Si l’encodage n’est pas indiqué au début du document, il est déterminé à l’aide de la commande file. Lorsqu’il est possible d’identifier l’encodage, la page est alors convertie en UTF-8.

Pour extraire le contenu textuel brut, la commande lynx -dump est utilisée.

Pour rechercher le code de réponse HTTP, l’encodage de la page et les phrases contenant le mot « сердце », on utilise la commande egrep en combinaison avec des expressions régulières, ainsi que la commande sed, qui permet de supprimer les parties inutiles des résultats de recherche. Lors de la recherche des occurrences du mot « сердце », l’expression régulière prend en compte toutes ses terminaisons possibles : сердц(е|а|у|ем|ец|цами|цах).

Pour compter le nombre de lignes contenant le mot « сердце » sur l’ensemble des pages analysées, la commande wc -l est utilisée.

Pour la tokenisation du contenu textuel des pages web et l’application ultérieure des scripts PALS, un script Python a été créé, utilisant le module de traitement du langage naturel Spacy.

Le nuage de mots représentant les termes apparaissant fréquemment avec le mot « сердце » a été généré à l’aide du module Python WordCloud.

UTILISATION DU PORTUGAIS

Le script peut être exécuté à l’aide de la commande suivante : 

bash script_pt.sh ../URLs/urls-pt.txt ../tableaux/tableau-pt.html

Le script prend en compte les variations de « сoração » en utilisant une expression régulière (coração|coracao|corações|coracoes) qui capture toutes les formes possibles du mot, singulier et plurielle, qu'elles soient accentuées ou non. Cette approche permet de comptabiliser correctement les occurrences même dans les textes où l'encodage des caractères spéciaux pourrait poser problème.

Dans ce programme, la commande curl est utilisée pour télécharger la structure HTML des pages web sélectionnées pour le corpus. Après le téléchargement, l’encodage du texte est vérifiée et lorsqu’il est possible l'identifier, la page est convertie en UTF-8.

Pour la nettoyage du contenu, l'extraction du texte brut, on utilise la commande lynx -dump, suivi de plusieurs filtres sed, a fin de supprimer les références, boutons, et contenus non-textuels (JSON, XML, oEmbed), qui ont été trouvées dans les textes et que sont inutiles pour la recherche.

Pour la recherche du mot et de ses variantes, la commande egrep est utilisée en combinaison de l'expression regulière. Pour isoler chaque occurrence individuelle et compter le nombre de lignes, on utilise la commande wc -l.

Le script structure automatiquement les résultats dans les dossiers : aspirations/ (pages HTML brutes), dumps-text/ (versions texte nettoyées), contextes/ (lignes contenant le mot-cible) et concordances/ (concordancier HTML). Il génère deux sorties HTML principales : un tableau de synthèse qui liste toutes les URLs analysées avec leurs métadonnées et statistiques, et un concordancier détaillé qui présente chaque occurrence du mot dans son contexte immédiat.
